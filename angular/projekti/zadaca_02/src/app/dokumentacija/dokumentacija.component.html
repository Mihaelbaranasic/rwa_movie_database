Zadaci

Koraci Opis Status
1. Na stranicu dokumentacija ažurirati tablice iz prve zadaće oko informacija koji dijelovi su napravljeni za servis odnosno aplikaciju. Ako je nešto napravljeno dijelomično treba to jasno napisati koji dio je napravljen. Ako ovog dijela nema bodovi vezani uz aplikaciju i/ili web servis nose 0 bodova. Tablice sa prve zadaće su ažurirane.
2. Na stranicu dokumentacija kreirati tablicu po koracima (počevši od zadaci općenito) ovog dokumenta u kojoj jasno pišu elementi opisa 2 zadaće koji su napravljeni, koji nisu. Ako ovog dijela nema zadaća nosi 0 bodova. Može se napraviti još jedna tablica sa dodatnim opisima ako se želi nešto naglasiti. Nije obavezno. Napravljena je tablica za drugu zadaću.
3. Složite package.json tako da se sa npm start može pokrenuti server i isto tako jedan package.json, tsconfig.json i angular.json da se može pokrenuti angular projekt. Napravljeno
4. Kod predavanja zadaće izbrisati svoje ključeve za TMDB servis iz konfiguracijske datoteke svaki student ubacuje svoje ključeve u konfiguracijsku datoteku kod testiranja. Napravljeno
5. Klientski dio Web Aplikacije HTML, CSS i JS na strani klijenta prebaciti u Angular sa TypeScript-om. Svi klijentski dijelovi Web Aplikacije su prebačeni u Angular.
6. Pozadinski dio web aplikacije poslužuje kompajliranu verziju Angular aplikacije. Moguće je pokretanje Angular aplikacije preko live servera sa ng serve, ali ta varijanta nosi manje bodova Pozadinski dio web aplikacije poslužuje kompajliranu verziju Angular aplikacije.
7. Pozadinski dio web aplikacije poslužuje dokumentaciju - moguće je dokumentaciju ugraditi u samu angular aplikaciju Dokumentacija je ugrađena u angular aplikaciju.(u dokumentaciji u folderu server se nalaze samo slike autora i ERA modela i skripta.sql)
8. Pozadinski dio web aplikacije nudi podršku u obliku REST servisa. REST servis sadrži sve metode kako je to bilo u prvoj zadaći za rad s bazom podataka. Dodatno svi ostali pozadinski dijelovi prve zadaće (npr. komunikacija sa TMDB servisom) koji nemaju direktne veze s bazom se sada tretiraju kao proširenje REST servisa jer klijentski dio aplikacije sada mora slati sve preko fetch zahtjeva. Angular aplikacija nikad ne komunicira direktno sa TMDB servisom već preko pozadinskog dijela kao u prvoj zadaći. Angular komunicira sa TMDB samo preko servera. Server sadrži sve metode kao i u prvoj zadaći, te samo dolazi do problema kod korištenja jwt-a kod dodavanja serije u bazu. (mapa detalji-serije)
9. Sada svi pozadinski dijelovi osim inicialnog posluživanja Angular aplikacije i prijave traže validan JWT. Kao što je i opisano pozadinski dijelovi traže validan JWT.
10. Informacije o bazičnoj adresi i portu za pozadinski dio koji koristi Angular aplikacija kod slanja fetch zahtjeva čitaju se iz environment datoteke. Hard kodirana varijanta nosi manje bodova. Sve se čita it environment datoteke.
11. Potrebno je doraditi dizajn stranice ako u prvoj zadaći niste vodili o tome previše računa. Dizajn treba prebaciti korištenjem Sass preprocesora u Sass ili Scss sintaksi. Čisti CSS ili bootstrap varijante nose 0 bodova. Sav css prebačen je u Scss sintaksu
12. Dodati reCaptcha v3 sustav na minimalno 3 obrazaca od kojih su obavezni: prijava, registracija i ažuriranje profila. Nije napravljeno
13. Potrebno je doraditi ažuriranje profila sa opcijom da se uključi dvorazinska autentifikacija. Prilikom prvog uključivanja dvorazinske autentifikacije potrebno je kreirati tajni ključ koji se sprema u bazu i prikazuje korisniku utekstualnom obliku ili u obliku QR koda (za dodatne bodove). Korisnik može isključiti dvorazinsku autentifikaciju ili ponovo uključiti, ali više nema prikaza tajnog ključa. Nije napravljeno
14. Potrebno je doraditi prijavu da se autentifikacija radi u dvije razine. Prvo korisničko ime i lozinka te nakon toga TOTP ako je uključena dvorazinska autentifikacija za tog korisnika. Nije napravljeno
15. Potrebno je doraditit prijavu da omogućuje mogućnost prijave sa oAuth putem GitHub računa. Takav korisnik nema TOTP. Nije napravljeno
16. Korisnik koji se prijavi preki GitHub računa ne radi ništa s bazom podatka. Kod takvog korisnika spremanje favorita se radi u preglednikov local storage. Jednako tako pregled favorita i favoritDetalji se čitaju iz local storage u pregledniku. Stranica profila nije dostupna. Nije napravljeno
17. Inicijalno mora postojati korisnik iz svake uloge koji nema TOTP i treba navesti podatke na stranici prijava radi lakšeg testiranja. Navedeni su svi korisnici, te nijedan korisnik ne koristi TOTP.
18. Za dodatne bodove može se prebaciti cijeli pozadinski dio u TypeScript. Kod pokretanja sa npm start obavezno se mora napraviti recompile TypeScript-a i kopiranje svih bitnih statičnih sadržaja da aplikacija normalno radi. Nije napravljeno
